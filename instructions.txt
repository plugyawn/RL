1000-step server validation for stochastic row-sampled Kondo

Scope
- Validate the current "groupfix" Kondo implementation against plain sync GRPO on a stable, already-tested recipe.
- This is a server validation run, not a hyperparameter sweep.
- Hold the recipe fixed and compare only:
  - baseline GRPO
  - stochastic Kondo ("groupfix")

Why this exact recipe
- Model: Qwen/Qwen2.5-1.5B-Instruct
- Task/env: GSM8K train with GSM8K test evaluation
- Runtime:
  - sequence_packing=false
  - dynamic_batching=true
  - single-node, single-GPU learner path
- This is the combo that has already been exercised repeatedly in this branch and avoids the unstable/unsupported paths we hit with sequence packing and worker-env rebuilds.

Recommended server
- Preferred: 1x A100 80GB
- Acceptable: equivalent single-GPU server with enough VRAM to run the exact same recipe without changing batching or model length

Required environment
- Export:
  - NEMO_RL_PY_EXECUTABLES_SYSTEM=1
- Use the checked-in NeMo-RL worktree directly; do not mix with an older wheel or another patched copy.

Run matrix
- Minimum:
  - seeds 42, 43, 44
  - baseline + groupfix
- Preferred:
  - seeds 42, 43, 44, 45, 46, 47
  - baseline + groupfix

Fixed recipe
- Steps: 1000
- Validation period: 50
- Validation at end: true
- Max validation samples: 256
- Prompts per step: 2
- Generations per prompt: 4
- Train global batch size: 8
- Train micro batch size: 1
- Logprob batch size: 1
- Max total sequence length: 768
- Generation max new tokens: 320
- vLLM max model len: 768
- Checkpointing: disabled for throughput unless the server harness requires it

- Run from repo root.

Groupfix command delta
- Same command as baseline, but replace the last line with:

  grpo.kondo.enabled=1 \
  grpo.kondo.mode=stochastic_response_rows \
  grpo.kondo.target_backward_token_fraction=0.7 \
  grpo.kondo.priority_mode=split_dual_tempered_delight \
  grpo.kondo.surprisal_temperature=0.1 \
  grpo.kondo.positive_keep_floor=0.95 \
  grpo.kondo.negative_keep_floor=0.25 \
  grpo.kondo.min_keep_probability=0.05 \
  grpo.kondo.bypass_on_nonfinite=true

What to archive
- Full log dirs for every run
- All train_data_step*.jsonl
- All val_data_step*.jsonl
- TensorBoard event files
- Exact command.txt per run

What to report back
- Per-seed final GSM8K test reward_mean
- Mean final reward_mean across seeds
- Validation curves at every saved checkpoint
- Mean policy_training wall time
- Mean actual backward fraction for groupfix
- Mean realized compute skip for groupfix
- Win/loss count by seed
- Whether the method still shows mid-horizon lead and any late-horizon fade

Decision rule
- This validation is a success only if groupfix keeps a real compute reduction and remains competitive at long horizon.
- If it only works by drifting back toward dense training, reject it as a Kondo result.
